# ملاحظات تنفيذ Codex

آخر تحديث: 2026-06-28

## الهدف

تم توحيد ربط البيانات الدراسية عبر النظام حول مسار واحد:

`المرحلة -> الفصل -> الشعبة -> الطالب / المادة / الاختبار / الرسوم / النقل`

الواجهة تعرض "الفصل" فقط بدلاً من الخلط بين "الصف" و"الفصل"، بينما يبقى الربط الداخلي دقيقاً عبر المرحلة والفصل والشعبة.

## الملفات الأساسية المعدلة

- `src/contexts/StageContext.tsx`
  - تعريف هيكل المدرسة الثابت: رياض الأطفال 2، الابتدائي 6، المتوسط 3، الثانوي 3.
  - إضافة دوال ضبط الفصول المسموحة حسب المرحلة.

- `src/lib/school-structure.ts`
  - دوال مركزية للربط والتحقق:
    - `getGradesForStage`
    - `isItemAllowedForGrade`
    - `getSectionClassId`
    - دوال مساعدة للفصول والشعب.

- `src/contexts/GlobalStoreContext.tsx`
  - فرض سلامة البيانات عند الحفظ وليس في الواجهة فقط.
  - عند تحديث الطالب أو ربطه بشعبة، يتم اعتماد المرحلة والفصل من الشعبة.
  - منع حفظ مادة خارج فصول المرحلة.
  - منع إسناد تدريسي إذا كانت المادة أو المعلم أو الشعبة لا تطابق المسار.
  - منع إنشاء اختبار أو حفظ درجة إذا لم يطابق الطالب والاختبار نفس المرحلة والفصل والشعبة والمادة.
  - اشتراك النقل يولد أو يحدث فاتورة نقل تلقائياً في المالية.

- `src/routes/exams.index.tsx`

  - إزالة حقل الفصل الدراسي من أسفل النموذج.
  - إزالة التكرار بين الصف والفصل، واعتماد تسمية الفصل فقط.
  - القوائم تفتح تدريجياً: المرحلة ثم الفصل ثم الشعبة ثم المادة.

- `src/routes/exams.grades.tsx`
  - إدخال الدرجات حسب الفصل والشعبة والمادة والاختبار.
  - عرض الطلاب المطابقين لمسار الاختبار فقط.
  - إضافة بحث سريع داخل الطلاب.
  - حفظ الدرجات يرفض أي عدم تطابق بين الطالب والاختبار.

- `src/routes/exams.reports.tsx`
  - تقارير مجمعة وفردية بدون فصل دراسي ثابت.
  - الشهادة الفردية تعرض أسماء الاختبارات اليدوية ودرجاتها.
  - الفلاتر تعتمد على المرحلة والفصل والشعبة.

## المالية والمحاسبة

- `src/routes/finance.fees.tsx`
  - إعداد الرسوم حسب الفصول المعرفة في هيكل المدرسة.
  - الشعب المعروضة تتبع الفصول المختارة فقط.
  - حفظ الرسوم يرفض الشعبة غير المطابقة للفصل أو المرحلة.

- `src/routes/finance.discounts.tsx`
  - تحويل الخصومات من وصف عام إلى قواعد تطبيق فعلية.
  - الخصم يمكن أن يستهدف مرحلة، فصول، شعب، أو طلاب محددين.
  - عرض عدد الطلاب المطابقين حالياً لكل قاعدة خصم.
  - الخصومات غير المفعلة لا تطبق على الفواتير الجديدة.

- `src/routes/finance.invoices.tsx`
  - إصدار الفواتير فردياً أو جماعياً حسب طالب أو شعبة أو فصل.
  - الفلاتر تعرض الفصل والشعبة مع بيانات الطالب الفعلية.
  - الفاتورة تعرض ولي الأمر والخصم المطبق.
  - منع تطبيق خصم يدوي على طلاب لا يطابقون نطاق الخصم.

- `src/routes/finance.payments.tsx`
  - تصحيح قيم طرق الدفع لتطابق الموديل.
  - فلاتر السندات حسب الفصل والشعبة.
